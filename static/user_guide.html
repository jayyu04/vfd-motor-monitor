01 操作流程
1
選擇模擬模式
在頂部控制列的下拉選單選擇要模擬的異常類型。開機前或開機後都可以切換，系統在進入 RUNNING 狀態後才會套用所選模式。
可選：NORMAL / OVERLOAD / STALL / LOAD_LOSS / BEARING_WEAR
2
按下「開機」
系統進入 STARTUP 啟動過渡期，持續約 3 秒。此期間電流較高為正常現象，Rules 和 ML 判斷自動暫停，不發出任何告警。VFD Header 顯示黃色「STARTUP」。
3
進入 RUNNING 狀態
啟動完成後進入穩態運轉，VFD Header 切換為綠色「RUNNING」。系統每 300ms 產生一筆資料，每 500ms 刷新畫面，異常判斷開始運作。
此時切換模擬模式可即時觀察告警變化
4
觀察即時數值與告警
監控頻率、電流、轉速、轉矩四個感測器數值，以及右側診斷面板的 Rule Score（0~100）和 ML 信心分數（%）。出現告警時查看最下方的告警記錄了解觸發原因。
Rule Score 越高代表異常越嚴重，ML 信心越高代表模型越確定
5
切換不同模式觀察差異
在 RUNNING 狀態下切換不同的模擬模式，可以觀察各種異常情境下感測器數值的變化，以及 Rules 和 ML 兩層判斷的結果差異。
BEARING_WEAR 需累積 5 筆資料後 Rules 才會觸發，ML 可更早識別
6
按下「關機」
系統停止產生資料，所有即時數值歸零，設備階段切換回 OFF。已記錄的歷史資料和告警紀錄保留在資料庫中，重新開機後仍可查看。
02 VFD Header 狀態說明
設備階段顏色代表什麼

VFD Header 右側的「設備階段」方框顏色會隨系統狀態即時變化，讓操作員一眼就能掌握目前的運轉狀況。

OFF
系統關機，馬達未運轉。所有感測器數值為零，Rules 和 ML 不判斷。
STARTUP
啟動過渡期（約 3 秒）。電流較高為正常現象，異常判斷自動暫停，不發出告警。
RUNNING
穩態運轉中。Rules 和 ML 雙層判斷運作，即時偵測異常，告警系統啟動。
模擬模式顏色

「模擬模式」方框的顏色反映目前選擇的異常類型，方便快速辨識當前模擬情境：

NORMAL 綠色 OVERLOAD 黃色 STALL 紅色 LOAD_LOSS 紅色 BEARING_WEAR 黃色
03 五種模擬模式說明
NORMAL　正常運轉
馬達在正常負載下穩定運轉，頻率和電流在正常範圍內小幅浮動。所有數值平穩，沒有異常跡象。
預期：Rule 0分 ／ ML NORMAL ／ 綜合 正常
OVERLOAD　過電流
機械負載過重，電流超過額定值（110% FLA 以上），頻率維持正常。電流數值偏高，進度條超過正常範圍。
預期：Rule WARNING ／ ML OVERLOAD ／ 綜合 警告
STALL　機械卡死
機械端堵轉，電流飆升，轉速被嚴重拖慢。電流極高、轉速明顯低於正常值，為最嚴重的異常。
預期：Rule CRITICAL ／ ML STALL ／ 綜合 緊急
LOAD_LOSS　負載流失
負載端斷裂（皮帶斷裂或幫浦空轉），馬達空載，電流降到極低值。頻率正常但電流接近零。
預期：Rule DANGER ／ ML LOAD_LOSS ／ 綜合 危險
BEARING_WEAR　軸承磨損
軸承早期磨損的預警模擬。頻率保持穩定，電流平均值正常，但電流出現規律性波動（漣波現象）。Rules 需累積 5 筆資料後才觸發，ML 可更早識別。趨勢圖上電流線會比 NORMAL 明顯更不穩定。
預期：Rule WARNING（累積後）／ ML BEARING_WEAR ／ 綜合 警告
04 儀表板各區塊說明
控制列
頁面最頂部。左側為開機／關機按鈕與模擬模式下拉選單。右側的「技術報告」和「使用說明」連結開啟對應文件（在新分頁中開啟）。
VFD Header
系統標題列。右側依序顯示：文件連結、設備階段（顏色方框）、當前模擬模式（顏色方框）、已運行秒數、ON/OFF 狀態指示燈。顏色跟隨狀態即時變化。
系統總覽
五個 KPI 卡片顯示累計統計：總資料筆數、警告筆數、危險筆數、緊急筆數、累計告警總數。數值隨每筆新資料持續更新。
感測器
左側四個卡片即時顯示最新一筆的頻率（Hz）、電流（A）、轉速（RPM）、轉矩（N·m）。每個卡片下方有進度條，視覺化數值大小。
診斷面板
右側面板顯示雙層判斷結果：設備階段、Rules 判斷類型與等級、ML 判斷類型與等級、Rule Score（0~100）、ML 信心分數（%）、綜合風險等級。綜合等級取 Rules 和 ML 兩者中較嚴重的結果。
趨勢圖
四個即時折線圖：① 頻率 Hz ＋ 轉速 RPM（虛線）、② 電流 A、③ 轉矩 N·m、④ Rule 分數 ＋ ML 信心（%）。最多顯示最近 200 筆資料，可觀察數值變化趨勢。
告警記錄
顯示最近 10 筆風險等級為警告以上的事件，包含時間、設備階段、模擬模式、頻率、電流、Rules 判斷、ML 判斷、信心分數、綜合等級。
完整資料表
最近 100 筆的完整資料，包含所有感測值與判斷結果。異常等級的列會有背景顏色標示（紅色為緊急，黃色為警告以上）。可橫向捲動查看所有欄位。
05 雙層判斷怎麼讀
Rules 和 ML 是兩件獨立的事

診斷面板同時顯示兩層判斷結果，它們是獨立運作的，各有不同的意義，不能混為一談。

Rule Score
0~100
分數 · 越高越嚴重
基於物理條件計算的異常嚴重程度。分數反映「現在這筆資料有多異常」，可以看到數值在慢慢爬升或下降。
0~14 → 正常
15~39 → 警告
40~64 → 危險
65~100 → 緊急
ML 信心分數
0~100%
百分比 · 越高越確定
隨機森林模型對這個判斷有多確定。87% 代表 100 棵決策樹中有 87 棵投票給這個類別。
100% → 完全確定
70~99% → 高度確定
50~69% → 有些把握
<50% → 落在模糊邊界
兩層結果如何合併

綜合風險等級取 Rules 等級和 ML 等級中較嚴重的那個。這樣設計的目的是讓兩層互補：

Rules 能快速抓到物理條件明確的異常，ML 能抓到規則式看不到的模糊邊界。只要其中一層發現異常，綜合等級就會反映出來。

範例一：Rule = NORMAL（0分）／ ML = WARNING（BEARING_WEAR 60%）
→ 綜合等級 = WARNING（ML 補位發現軸承磨損）

範例二：Rule = CRITICAL（78分）／ ML = CRITICAL（STALL 87%）
→ 綜合等級 = CRITICAL（兩層都確認緊急）
06 四個風險等級
正常
Rule Score 0 ~ 14
設備運作正常，持續監控，無需採取行動
警告
Rule Score 15 ~ 39
出現異常趨勢，安排檢查與維護，持續觀察變化
危險
Rule Score 40 ~ 64
立即檢查設備，考慮降載或計劃停機檢修
緊急
Rule Score 65 ~ 100
立即停機，避免設備損壞或發生人員安全事故
07 告警處理建議
緊急 STALL
機械卡死
馬達電流飆升且轉速被拖慢，機械端發生堵轉。繼續運轉會導致馬達線圈在短時間內過熱燒毀，是最危險的異常狀態。
處置：立即按下關機 → 檢查機械端是否有異物卡住、軸承損壞或傳動機構故障 → 排除原因後再重新啟動
危險 LOAD_LOSS
負載流失
馬達在高頻率下電流極低，負載端已斷裂或脫離。馬達空載長期高速運轉可能造成過速損壞，且設備無法正常運作。
處置：降速或停機 → 檢查皮帶是否斷裂、聯軸器是否脫落、幫浦是否空轉 → 確認負載端正常後再運轉
警告 OVERLOAD
過電流
電流持續超過額定值，機械負載過重。短時間內不會立即損壞，但長期過電流會導致馬達過熱，大幅縮短使用壽命。
處置：檢查負載是否過重 → 考慮降低負載或調整運轉頻率 → 觀察電流是否回到正常範圍 → 安排設備檢查
警告 BEARING_WEAR
軸承磨損預警
在固定頻率下電流出現異常波動，為軸承早期磨損的特徵。目前不需要立即停機，是預防性維護的提醒。若不處理，磨損持續惡化最終可能導致軸承卡死（STALL）。
處置：安排補充或更換軸承潤滑油 → 評估是否需要更換軸承 → 記錄發生時間追蹤惡化趨勢
