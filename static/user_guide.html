01 系統需求與啟動
系統需求

Python 3.9 以上，安裝以下套件：

streamlit　streamlit-autorefresh　pandas　scikit-learn　numpy

1
確認所有檔案都在同一個資料夾
config.py / VFD_simulator.py / comms.py / data_collector.py / physics.py / rules.py / ml_model.py / main.py / control.py / database.py / records.py / dashboard.py
static/ 資料夾裡要有 technical_report.html 和 user_guide.html
2
如果有舊的 pkl 檔，刪掉讓系統重新訓練
刪除 rf_model.pkl 和 rf_encoder.pkl，系統第一次啟動時會自動重新訓練。如果沒有這兩個檔案就跳過這步。
3
啟動系統
在終端機，進入專案資料夾後執行：streamlit run dashboard.py
瀏覽器會自動開啟，預設網址：http://localhost:8501
4
等待 ML 自動訓練完成
第一次啟動，系統會自動產生 5000 筆訓練資料並訓練隨機森林，約需 15~20 秒。終端機顯示「模型載入成功：rf_model.pkl」就代表完成，可以開始使用。
⚠️ 訓練期間請勿關閉終端機，等出現 dashboard 才算完成
⚠️ 第一次啟動很慢是正常的
系統在背景自動產生 5000 筆模擬訓練資料並訓練 100 棵決策樹，這個過程需要 15~20 秒。完成後 pkl 檔案會儲存下來，之後每次啟動只需要 1~2 秒載入，不需要重新訓練。
02 控制列完整說明
控制列位於頁面最上方，共有六個功能

這裡是所有操作的入口，所有指令都從這裡送出去。

開機
啟動馬達模擬。按下後系統開始每 300ms 產生一筆資料，VFD Header 顯示「啟動中」，5 秒後進入「運轉中」開始正常診斷。
關機
停止馬達模擬。資料停止產生，所有內部計算狀態清除，趨勢圖和資料庫保留已記錄的資料。
模式選單
選擇要模擬的工況：正常、過電流、機械卡死、負載流失、軸承磨損、自動測試。開機或關機狀態下都可以切換，不需要重新開機。
清除資料
清空資料庫、趨勢圖和告警記錄，重新開始。只有關機狀態才能按，運轉中按鈕是灰色的，這是防呆設計，避免意外清除正在記錄的資料。
技術報告
開啟系統技術報告，說明系統架構、物理計算、Rules 和 ML 的設計原理，適合深入了解系統。
使用說明
開啟本說明文件（你現在正在看的這個）。
💡 建議操作流程
1. 選好模式 → 2. 按開機 → 3. 等 5 秒啟動遮蔽結束 → 4. 觀察診斷結果

切換模式不需要重新開機，直接在選單切換，下一筆資料（約 300ms）就會反映新的工況。
03 六種模擬模式
每種模式模擬真實工業場景中的不同狀況

選擇模式後，VFD_simulator 會產生對應的頻率、電流訊號。Rules 和 ML 不知道你選了什麼，純粹根據訊號特徵做判斷。

NORMAL
正常運轉
頻率：30~60 Hz　電流：10.5~14 A
正常工況，兩層診斷都應判為正常。用於確認系統在正常狀態下不會誤報。頻率範圍廣是因為操作員可以依需求設定變頻器輸出頻率（30Hz 低速、60Hz 高速）。
OVERLOAD
過電流
頻率：30~60 Hz（任何操作頻率）　電流：18.5~22 A（110%+ FLA）
任何操作頻率下電流超過額定值，模擬機械端過載。觸發即為 WARNING 等級。
BEARING_WEAR
軸承磨損
頻率：30~60 Hz（幾乎穩定）　電流：8~15 A（中心值正常，但大幅振盪）
任何操作頻率下電流中心值正常但電流振幅大，模擬軸承磨損產生的電流漣波。需要累積 20 筆資料（約 6 秒）計算標準差，Rule 才觸發。切換後約 6 秒才出現 Rule 告警，但 ML 會提早補位顯示 WARNING。應觸發 WARNING 等級。
LOAD_LOSS
負載流失
頻率：30~60 Hz（任何操作頻率）　電流：2~4 A（極低）
任何操作頻率下電流極低，模擬皮帶斷裂或幫浦空轉。應觸發 DANGER 等級。馬達在空轉，沒有做任何有用的功，長期空轉也會造成散熱問題。
STALL
機械卡死
頻率：20~35 Hz（拉不上去）　電流：24~30 A（160%+ FLA）
電流飆高且頻率拉不上去，模擬機械端堵轉。觸發即為 CRITICAL，是最嚴重的異常。現實中繼續運轉幾分鐘就會燒毀馬達線圈，應立即停機。
AUTO
自動測試
每 10~20 秒自動隨機切換工況（不重複上一個）
系統自動切換工況，Rules 和 ML 在不知道工況的情況下自己判斷。VFD Header 即時顯示目前實際工況，趨勢圖下方出現對比表格，可以即時驗證 ML 準不準確。這是展示 ML 真實能力最有說服力的方式。
04 VFD Header 狀態列
位於控制列正下方，即時顯示設備狀態

每 500ms 更新一次，顯示系統目前的即時狀況。

設備階段
machine state
關機：系統未啟動，沒有資料產生
啟動中：開機後前 5 秒，電流偏高是正常的啟動現象，不做異常診斷
運轉中：正常監控狀態，每 300ms 診斷一次並寫入資料庫
模擬模式
current fault
顯示目前實際模擬的工況名稱。
手動模式：顯示你在選單選的工況。
AUTO 模式：直接從 VFD_simulator 讀取系統自動切換到的實際工況，讓你知道現在是哪個工況，方便跟 ML 判斷對比。
已運行
elapsed time
從按下開機到現在的秒數，持續累積。
前 5 秒為啟動期，不做診斷。5 秒後開始正常診斷。
綜合等級
final level
最新一筆資料的綜合風險等級，取 Rules 和 ML 兩者較嚴重的那個。
正常 / 警告 / 危險 / 緊急
⚠️ 開機後 5 秒診斷結果都是「啟動中」，分數都是 0，這是正常的
馬達開機時電流是額定的 1.5~2 倍（約 22~30A），這是變頻器限流啟動的正常現象，不是異常。如果不遮蔽，高啟動電流會被誤判為緊急告警。5 秒後電流降回正常範圍，系統才開始正常診斷。這 5 秒的資料在資料庫裡會標記為「啟動中」。
05 儀表板各區塊說明
① 最近工況流程圖（頁面最上方）

顯示最近三個不重複的工況變化，用箭頭串連，讓你一眼看出系統最近的歷史。例如：正常 → 機械卡死 → 正常。

方框顏色對應風險等級。最右邊（最新）的方框最亮，舊的稍微透明。只有在有資料的情況下才會出現。

② 系統總覽（5 個 KPI 卡片）

顯示目前累積資料的統計摘要：總資料筆數、警告筆數、危險筆數、緊急筆數、最新記錄時間。讓你快速掌握整體狀況，不用翻資料表。

③ 感測器即時數值（4 個卡片）

顯示最新一筆資料的四個即時數值，每 500ms 更新：

頻率 Hz：變頻器輸出頻率，直接感測器量測值，最可靠
電流 A：馬達電流，直接感測器量測值，最重要的監控指標
轉速 RPM：同步轉速，用公式 120×Hz/4 計算的理論值，不是實際轉速
轉矩 N·m：輸出轉矩，用功率和轉速推算的理論值

每個卡片下方有進度條顯示相對於最大值的比例。

④ 診斷面板（最核心的資訊）

這是系統的核心輸出，分左右兩欄加下方：

左欄 — Rules 判斷：Rule Confidence（0~100%）、判斷的異常模式、風險等級、觸發的具體原因（例如「電流 27.0A > 20.0A (133% FLA) [70%]」）
右欄 — ML 判斷：隨機森林判斷的異常類型、風險等級、信心分數（100棵樹中有幾%投這個結果）
下方 — 綜合等級：取兩者較嚴重的，這是最終的告警等級

⑤ 感測器趨勢圖（最近 200 筆）

四個折線圖，顯示最近 200 筆資料的變化趨勢，可以看出異常是突然發生還是漸進惡化：

頻率 / 轉速：STALL 時頻率明顯下降；其他異常頻率通常正常
電流：OVERLOAD 和 STALL 時電流飆高；LOAD_LOSS 時極低；BEARING_WEAR 時電流大幅振盪（振幅遠大於 NORMAL）
轉矩：跟電流連動，STALL 時極高，LOAD_LOSS 時極低
Rule Confidence / ML 信心：信心分數的變化趨勢可以看出異常是否在惡化

⑥ AUTO 模式即時驗證（只在 AUTO 模式出現）

選擇 AUTO 模式後，趨勢圖下方出現這個對比表格，顯示最近 10 筆資料：

時間：這筆資料的記錄時間
實際工況：VFD_simulator 目前真正在模擬的工況（正確答案）
ML 判斷：ML 在不知道工況的情況下，根據訊號特徵判斷的結果
ML 信心：模型對這個判斷有多確定
正確：✅ 代表 ML 判對了，❌ 代表判錯了

右上角的 ML 準確率 是最直接展示 ML 真實能力的指標。

⑦ 最近異常事件（告警記錄）

只顯示 WARNING 以上等級的記錄，讓你快速找到需要關注的時間點。包含時間、設備階段、頻率、電流、Rules 判斷、ML 判斷、ML 信心、綜合等級。

⑧ 完整資料表（最近 100 筆）

顯示最近 100 筆所有資料，包含完整欄位：時間、設備階段、頻率、電流、轉速、轉差率、轉矩、Rules 結果、ML 結果、綜合等級。適合做事後分析或查看特定時間點的詳細資料。

06 雙層判斷怎麼讀
Rules 的 Rule Confidence 怎麼看

Rule Confidence 越高代表 Rules 對這個模式的判斷越確定。診斷面板的觸發原因說明了為什麼這個模式被觸發，完全透明可解釋。

ML 的信心分數怎麼看

90%+：模型非常確定，高度可信，通常準確
70~90%：有把握，通常正確
50~70%：落在模糊地帶，建議同時參考 Rules 的判斷

信心低不代表 ML 判錯了，可能是資料真的落在兩種異常的交界地帶，這是雙層判斷互補最有價值的情境。

✅ 兩層一致

Rules 和 ML 判斷相同，可信度最高。例如兩層都說 STALL CRITICAL，代表這個異常特徵非常明顯，可以直接採取行動。

⚠️ 兩層不一致

最常出現在模糊地帶。綜合等級取較嚴重的，建議持續觀察趨勢，並參考 Rules 的觸發原因來判斷實際狀況。

💡 三個 ML 補位的具體場景
場景一：OVERLOAD 輕度（電流 17~19A）
Rules 信心只有 4~22%，不足以確認是 OVERLOAD，判 NORMAL。ML 從電流比例和轉矩組合識別出 OVERLOAD → final = WARNING

場景二：BEARING_WEAR 前 20 筆（窗口未填滿）
Rules 需要 20 筆才能計算電流標準差，前 19 筆信心 0% 判 NORMAL。ML 從單筆電流振幅識別 → final = WARNING（提早約 6 秒）

場景三：兩層都確認（最可信）
STALL 電流 28A：Rules 信心 80%，ML 信心 100%，兩層都判 STALL CRITICAL → final = CRITICAL（高度可信）
07 四個風險等級與處置建議
正常
Score 0 ~ 24
持續監控即可，無需採取任何行動
警告
Score 25 ~ 49
安排人員檢查，持續觀察分數趨勢是否惡化
危險
Score 50 ~ 74
立即檢查設備，考慮降載或計劃近期停機檢修
緊急
Score 75 ~ 100
立即停機，繼續運轉可能造成設備損壞或人員危險
各工況設計對應的等級

STALL（機械卡死） → 緊急 CRITICAL → 立即停機
LOAD_LOSS（負載流失） → 危險 DANGER → 盡快處理
OVERLOAD（過電流） → 警告 WARNING → 安排檢查
BEARING_WEAR（軸承磨損） → 警告 WARNING → 預防性維護

這個對應關係是刻意設計的，確保嚴重程度和風險等級完全一致。

08 常見問題
Q：第一次啟動要等很久，終端機一直在跑，正常嗎？
正常。系統在自動產生 5000 筆訓練資料並訓練隨機森林（100棵決策樹），大約需要 15~20 秒。看到終端機顯示「模型載入成功：rf_model.pkl」就代表完成了。之後每次啟動只需要 1~2 秒直接載入，不需要重新訓練。
Q：開機後 5 秒，診斷結果都是「啟動中」，分數都是 0，這是正常的嗎？
完全正常，這是設計行為，叫做「啟動遮蔽」。馬達開機時電流是額定的 1.5~2 倍（約 22~30A），是正常的限流啟動現象。如果不遮蔽，這高電流會被誤判為緊急告警。5 秒後電流降回正常範圍，系統才開始正常診斷。
Q：切換到 BEARING_WEAR 模式，等了幾秒才出現告警，正常嗎？
正常。軸承磨損的 Rules 判斷需要累積最近 20 筆資料計算電流標準差，每筆間隔 300ms，所以需要約 6 秒。但 ML 不需要等窗口，從第一筆特徵組合就能識別，所以 final 等級會在 6 秒內就顯示 WARNING（ML 補位）。Rule 告警要等 6 秒才出現是正常的。
Q：AUTO 模式下，VFD Header 顯示的工況和 ML 判斷不一樣，哪個是對的？
VFD Header 顯示的是「實際工況」，也就是 VFD_simulator 目前真正在模擬的工況，這是正確答案。ML 判斷是系統根據訊號特徵推測的結果。兩者不一樣就代表 ML 這次判錯了（對比表格會顯示 ❌）。AUTO 模式的對比表格就是用來即時驗證 ML 準不準確的。
Q：清除資料按鈕是灰色的，按不下去？
這是防呆設計。清除資料只有在關機狀態下才能按，避免在運轉中意外清掉正在記錄的資料。請先按「關機」，等 VFD Header 顯示「關機」狀態後，再按「清除資料」。
Q：切換模式需要重新開機嗎？
不需要。直接在選單切換模式，下一筆資料（約 300ms 後）就會反映新的工況。系統會即時切換，不需要關機再開機。
Q：Rules 和 ML 判斷不一樣，哪個比較準？
視情況而定。特徵明顯的異常（STALL、LOAD_LOSS）兩層通常一致。在輕度 OVERLOAD、STALL 低電流模糊地帶、BEARING_WEAR 初期這三個場景，ML 會補位。但 Rules 的觸發原因完全透明，不一致時可以參考 Rules 的觸發原因來輔助判斷。
Q：轉速顯示的是什麼？跟實際轉速一樣嗎？
顯示的是「同步轉速」，用公式 120×Hz÷4（極數）計算的理論值。這是馬達在這個頻率下的最高轉速，實際轉速因為轉差率（約 2~5%）會稍微低一點。系統沒有安裝轉速感測器，所以只能顯示理論同步轉速。
Q：趨勢圖看不到資料怎麼辦？
確認系統是否已開機且運轉中。趨勢圖只顯示有資料的部分。如果剛清除資料或剛開機，需要等幾秒資料累積後才會出現曲線。啟動期間（前 5 秒）的資料也會顯示在趨勢圖上，但診斷分數都是 0。
