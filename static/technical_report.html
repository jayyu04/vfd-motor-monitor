02 系統架構
十二個模組，每個只做一件事

系統採用嚴格分層設計，每個模組職責單一，不依賴其他模組的內部狀態。未來換真實硬體只需修改 comms.py 一個模組，換不同規格的馬達只需修改 config.py，其他模組完全不動。

config.py馬達銘牌規格、系統參數、Rules 閾值的集中管理，所有模組都來這裡讀取設定
records.py定義 FullRecord 資料結構，存放合併後的完整紀錄，避免 main.py 和 database.py 循環引用
VFD_simulator.py模擬真實變頻器，只輸出三個原始訊號：Hz / A / V，同時提供 get_current_fault() 讓 dashboard 即時讀取目前模擬的工況
comms.py通訊層，VFD 與電腦的橋樑。目前從 simulator 讀取，未來換 Modbus 只改這裡
data_collector.py接收原始訊號，加上 timestamp 和 motor_id，完成資料擷取
physics.py用物理公式從三個原始值推算：同步轉速、轉差率、轉矩
rules.py規則式分析層，計算 Anomaly Score（0~100），第一層判斷，結果回傳給 main.py
ml_model.py隨機森林機器學習，100棵決策樹投票，第二層判斷，結果回傳給 main.py
main.py流水線串接，判斷啟動遮蔽，合併 Rules + ML 結果，寫入資料庫
control.py系統狀態唯一管理者，接收 dashboard 指令，通知 main 開始/停止
database.pySQLite 資料庫，儲存完整紀錄，提供查詢介面給 dashboard
dashboard.py展示層，只跟 control.py 說話傳送指令，從 database.py 讀取資料顯示，從 VFD_simulator 讀取實際工況

04 VFD_simulator — 模擬資料產生
為什麼需要模擬器

真實工業監控需要接實體感測器，但開發和展示階段沒有真實硬體。VFD_simulator.py 的任務是產生「看起來像真實變頻器輸出」的資料，只輸出三個原始值：Hz / A / V，模擬真實硬體透過 Modbus 能直接讀到的最小資料集。

連續性設計：資料不會一下高一下低

真實感測器的數值不會每秒大幅跳動，而是在某個中心值附近小幅浮動。simulator 用「滑動狀態」實現這個特性：

第一筆：從工況的中心值範圍隨機取初始值
之後每筆：上一筆數值 + 小幅隨機漂移

# 範例：NORMAL 模式，頻率中心值 47.3Hz
47.3 → 47.5 → 47.2 → 47.4 Hz（平滑連續，不跳動）
三種運作狀態
OFF 狀態：
Hz=0 / A=0 / V=0，重置所有內部狀態

STARTUP（開機後 0~5 秒）：
頻率從 5Hz 線性爬升到目標頻率
電流是額定的 1.5~2 倍（22~30A）← 變頻器限流啟動，正常現象
隨啟動完成線性降回正常範圍

RUNNING（開機 5 秒後）：
依選擇的工況產生對應的穩態資料
六種工況設計

五種手動模式 + 一種自動模式，各模式的頻率和電流範圍刻意設計了重疊區間，讓 ML 能學到模糊邊界。

NORMAL　正常運轉
頻率：30~60 Hz（操作員設定範圍）
電流：10.5~14 A
頻率由操作員決定，電流反映負載大小，兩者獨立，符合 V/f 控制
OVERLOAD　過電流
頻率：30~60 Hz（任何操作頻率）
電流：18.5~22 A（110%+ FLA）
任何操作頻率下電流超過額定值，模擬機械過載。頻率不侷限於特定範圍，因為過載可以在任何轉速下發生
STALL　機械卡死
頻率：20~35 Hz（拉不上去）
電流：24~30 A（160%+ FLA）
電流飆高且頻率拉不上去，模擬機械端堵轉
LOAD_LOSS　負載流失
頻率：30~60 Hz（任何操作頻率）
電流：2~4 A（空載以下）
任何操作頻率下電流極低，模擬皮帶斷裂或幫浦空轉。負載流失可以在任何轉速下發生
BEARING_WEAR　軸承磨損
頻率：30~60 Hz（任何操作頻率）
電流：10~13 A（正常，但波動 ±2.5A）
任何操作頻率下電流中心值正常但波動幅度大，模擬軸承磨損產生的電流漣波
AUTO　自動隨機切換
每 10~20 秒自動切換工況
BEARING_WEAR 持續 20~30 秒
系統自動切換，用於驗證 ML 在不知道答案的情況下能否正確判斷
05 物理計算層 — 從三個原始值推算物理量
為什麼要推算

真實變頻器透過 Modbus 只給出頻率、電流、電壓三個值，但監控和診斷需要更多有意義的物理量。physics.py 是系統裡唯一包含物理公式的地方，所有推算都在這裡完成。

三個核心公式
# 1. 同步轉速（理論最高轉速）
sync_rpm = 120 × frequency_hz / 極數
　　　　　= 120 × 50 / 4 = 1500 RPM（額定頻率下）

# 2. 轉差率（從電流負載比估算）
load_ratio = current_a / RATED_CURRENT_A
slip_ratio = 0.02 + load_ratio × 0.05
　※ 沒有轉速感測器，用電流近似估算，是估計值

# 3. 轉矩（理論值）
P_in = √3 × V × I × cosφ
P_out = P_in × η
ω = 2π × sync_rpm / 60
T = P_out / ω
直接量測值（可信）

frequency_hz
current_a
voltage_v

這三個直接來自感測器，最可靠。

估算值（理論值）

sync_rpm
slip_ratio
torque_nm

用公式推算，沒有轉速感測器，屬於理論估計值。

06 Rules 判斷 — 規則式異常評分
核心概念：用分數取代是非題

傳統做法是直接判斷「正常」或「異常」，沒有灰色地帶。我們改用 Anomaly Score（0~100分），分數越高代表越嚴重，讓判斷有層次感。操作員不只知道「有沒有問題」，還能看到「分數在慢慢爬升」，提早做出應對。

白話比喻
就像體溫計，不只告訴你「有沒有發燒」，而是告訴你「現在 38.5°C，比昨天的 37.8°C 高了」。分數的變化趨勢比單純的是非判斷更有價值。
分數與風險等級對應
正常
0~14
警告
15~39
危險
40~64
緊急
65~100
四個條件如何計分

四個條件各自獨立計分，最後取分數最高的那個作為最終結果。

STALL（機械卡死） → 最高 84 分 → 緊急

觸發條件：電流 > 20A（133% FLA）且 轉差率 > 12%
電流每超過 20A 一安培 → +8 分（最多 48 分）
滑差每超過 12% 一個百分點 → +6 分（最多 36 分）
兩個條件都滿足才給滿分，單一條件只給一半，避免誤報

LOAD_LOSS（負載流失） → 最高 64 分 → 危險

觸發條件：頻率 > 40Hz 且 電流 < 5A
score = (5 - 電流) / 5 × 30 + 40，上限 64 分
兩個條件必須同時成立，避免低電流單獨誤報

OVERLOAD（過電流） → 最高 29 分 → 警告

觸發條件：電流 > 16.5A（110% FLA）
每超過 1A → +6 分，上限 29 分（限制在 WARNING 區間）
上限設計目的：過電流是警告，不讓它跑到危險等級

BEARING_WEAR（軸承磨損） → 最高 39 分 → 警告

觸發條件：電流標準差 > 0.8A 且 頻率變化 < 0.5Hz
需累積最近 20 筆資料計算標準差（滑動窗口，約 6 秒）
上限 39 分（限制在 WARNING 區間）
頻率必須穩定，排除頻率變化本身造成的電流波動
啟動遮蔽

馬達開機時電流是額定的 1.5~2 倍，這是正常現象不是異常。main.py 在開機後 5 秒內不呼叫 Rules，避免把正常的高啟動電流誤判為緊急告警。

07 ML 判斷 — 隨機森林機器學習
先理解：什麼是監督式學習

機器學習有很多種，我們用的是「監督式學習」。監督式學習的核心概念：給機器看大量「有正確答案的例題」，讓它自己找出規律，之後遇到新題目就能判斷。

白話比喻
就像教小孩認動物：給他看 1000 張貓的照片說「這是貓」，再給他看 1000 張狗的照片說「這是狗」。學習完之後，看到新照片就能自己判斷。我們的 ML 也是這樣，只不過學的是「這是 STALL」「這是 NORMAL」。
訓練資料怎麼來

我們沒有真實的歷史異常資料，所以用 VFD_simulator 批次產生訓練資料。每種工況產生 1000 筆，五種工況共 5000 筆，每筆都附有正確答案（label）。

NORMAL × 1000 筆 → 讓模型學習正常運轉的特徵
OVERLOAD × 1000 筆 → 讓模型學習過電流的特徵
STALL × 1000 筆 → 讓模型學習機械卡死的特徵
LOAD_LOSS × 1000 筆 → 讓模型學習負載流失的特徵
BEARING_WEAR × 1000 筆 → 讓模型學習軸承磨損的特徵
──────────────────────────────
總計 5000 筆，打亂順序後分割：
訓練集 4000 筆（80%）← 拿來學習
測試集 1000 筆（20%）← 拿來考試，模型沒看過
訓練集 vs 測試集：為什麼要分開

如果用同一批資料訓練又評估，就像用考古題練習然後用同一套考古題考試，100 分沒有意義。測試集是模型從來沒看過的資料，用它評估才能知道模型真正學會了多少。

我們的測試集準確率：100%，代表模型能正確區分五種異常模式。

決策樹是怎麼學習的

隨機森林是由很多棵「決策樹」組成的。先理解一棵決策樹怎麼學習，就能理解整個隨機森林。

訓練的時候，決策樹會從 4000 筆訓練資料裡，自動找出最能區分五種工況的問題，一層一層往下問，長出一棵樹狀判斷結構：

根節點：「電流超過 20A 嗎？」
├── 是 → 「滑差超過 12% 嗎？」
│　　　　├── 是 → STALL（機械卡死）
│　　　　└── 否 → OVERLOAD（過電流）
└── 否 → 「電流低於 5A 且頻率 > 40Hz 嗎？」
　　　　　├── 是 → LOAD_LOSS（負載流失）
　　　　　└── 否 → 「電流標準差 > 0.8A 嗎？」
　　　　　　　　　　├── 是 → BEARING_WEAR（軸承磨損）
　　　　　　　　　　└── 否 → NORMAL（正常）

每個節點問的問題，是演算法從 6 個特徵裡自動找出「最能把資料分乾淨的切割點」，不是人工設定的。

為什麼要 100 棵樹而不是 1 棵
一棵樹容易「記死答案」——它對訓練資料學得太好，但換一筆稍微不同的資料就可能判錯。100 棵樹的解法是：每棵樹訓練時隨機使用不同的資料子集和特徵組合，讓每棵樹各自走不同的路。就算幾棵樹判斷錯了，大多數還是對的，投票結果更可靠。就像問 100 位專家，多數決比一個人的判斷更穩定。
決策樹怎麼「長出來」

隨機森林裡的每一棵決策樹，都是從訓練資料裡學出來的，不是人工寫的規則。學習過程如下：

步驟一：隨機抽取約 3200 筆訓練資料（不用全部 4000 筆）
步驟二：在根節點，隨機選幾個特徵來考慮
步驟三：找出哪個問題能讓分出來的兩群「最純粹」
　例如：「torque_nm > 80？」
　　→ 是：這群幾乎都是 STALL（純粹）
　　→ 否：這群混著 NORMAL / OVERLOAD（不純）
步驟四：對每一群繼續問問題，一層一層往下
步驟五：直到某群「只剩一種類別」就停下來，這裡叫葉子節點
步驟六：這棵樹長好了，存起來

重複以上步驟 100 次，每次都隨機抽不同的資料和特徵，就長出 100 棵長相不同的樹。

白話比喻：為什麼每棵樹要隨機
如果 100 棵樹都用完全一樣的資料和特徵，長出來會完全一樣，投票就沒有意義了。

隨機性讓每棵樹從不同角度學習：
樹001 主要看 torque 和 current → 擅長判斷 STALL
樹002 主要看 slip 和 frequency → 擅長判斷 LOAD_LOSS
樹003 主要看 current_std_5 → 擅長判斷 BEARING_WEAR

就像三個專家各有所長，投票結果比任何一個人單獨判斷更可靠。
投票機制與信心分數
新資料進來：Hz=25, A=24, slip=0.15, torque=85...

樹001 → STALL ／ 樹002 → STALL ／ 樹003 → OVERLOAD
...（共 100 棵）

投票結果：STALL 87票 ／ OVERLOAD 13票

最終答案：STALL
信心分數：87%（87票 / 100棵）

信心分數低（50% 左右）代表資料落在模糊地帶，模型不確定，這時候 Rules 的判斷更有參考價值。

六個輸入特徵

模型不直接看原始感測值，而是使用 6 個有物理意義的特徵：

01
torque_nm
轉矩（最重要特徵）。STALL 時極高，LOAD_LOSS 時極低，區分度最大
02
current_a
電流絕對值，最直接反映負載狀態的感測器數值
03
current_ratio
電流 / 額定電流，正規化後的相對負載比，比絕對值更穩定
04
slip_ratio
轉差率，STALL 時超過 12%，是判斷機械卡死的關鍵指標
05
frequency_hz
頻率，STALL 時拉不上去，LOAD_LOSS 時正常，提供背景資訊
06
current_std_5
最近 20 筆電流的標準差，BEARING_WEAR 的唯一關鍵特徵，捕捉電流漣波。20 筆（約 6 秒）讓標準差更穩定，誤報更少
為什麼選隨機森林
✅ 小資料集（5000筆）就能訓練得很好，神經網路需要更多
✅ 不需要正規化，特徵單位不同（Hz、A、N·m）不影響
✅ 可解釋性強，特徵重要性可以輸出，知道模型在看什麼
✅ 不容易過擬合，100棵樹投票比單棵樹穩定
✅ 訓練快，工業場景資料量不大，幾秒就完成

⚠️ 不擅長捕捉時間序列的漸進趨勢（如電流慢慢爬升）
⚠️ 我們用 current_std_5 滑動窗口部分補償這個限制
08 雙層互補的價值
Rules 的強項

物理條件明確的異常，Rules 能快速給出結論，不需要累積資料，而且原因完全透明可解釋。

STALL：電流 26A + 滑差 17% → 84分 CRITICAL
立即告警，不需要等待
ML 的強項

多特徵組合的模糊地帶，ML 透過 100 棵樹的投票，比單一規則更精確。

BEARING_WEAR：Rules 需要 5 筆累積
ML 第一筆就能從特徵組合識別
最有價值的模糊地帶

NORMAL vs BEARING_WEAR

兩者：頻率正常、電流中心值相近（10~14A）
差別：電流波動大小（BEARING_WEAR 波動 ±2.5A，NORMAL 只有 ±0.3A）
Rules：需要累積 5 筆資料計算標準差才觸發
ML：從 current_std_5 特徵，更早識別出來

OVERLOAD vs STALL（電流 20~22A 的模糊地帶）

電流 21A 時兩個條件都會給分，Rules 難以區分
OVERLOAD 分數：27分 ／ STALL 分數：約 30 分（差距很小）
ML 看頻率 + 滑差 + 轉矩的組合
OVERLOAD 頻率正常，STALL 頻率低 → ML 能準確區分
綜合等級如何決定

取 Rules 等級和 ML 等級中較嚴重的那個。設計原則：寧可多報一次警告，也不要漏掉一次真正的異常。

Rules = NORMAL ／ ML = WARNING（BEARING_WEAR 65%）
→ 綜合 = WARNING（ML 補位，提早發現軸承磨損）

Rules = CRITICAL ／ ML = CRITICAL（STALL 87%）
→ 綜合 = CRITICAL（兩層都確認，高度可信）

正常
Score 0 ~ 14
設備運作正常，持續監控，無需採取行動

Score 15 ~ 39

各工況對應的設計等級
STALL → 緊急（Score 最高 84）← 立即停機
LOAD_LOSS → 危險（Score 最高 64）← 盡快處理
OVERLOAD → 警告（Score 最高 29）← 安排檢查
BEARING_WEAR → 警告（Score 最高 39）← 預防性維護

各工況的分數上限刻意設計成對應的等級，確保不同嚴重程度的異常落在正確的風險區間。

10 系統誠實限制與未來展望
現有的限制（誠實說明）
⚠️
訓練資料來自模擬器
訓練和測試資料都由同一個 simulator 產生，測試集準確率 100% 是在模擬資料分布上的結果。換成真實硬體資料後，準確率可能下降，需要用真實資料重新訓練或微調。這是 Simulation-to-Real 開發流程的標準限制。
⚠️
轉差率和轉矩是估算值
沒有轉速感測器，slip_ratio 是從電流負載比推算的近似值，torque_nm 是用同步轉速計算的理論值。加裝編碼器後才能得到真實的轉速和精確的轉矩。
⚠️
BEARING_WEAR 是簡化版
真實的軸承磨損偵測需要高頻取樣（1000Hz 以上）和 FFT 頻譜分析。我們用電流標準差做簡化版的特徵提取，適合 demo 展示，真實環境需要升級。
⚠️
STALL 雙條件實際上不完全獨立
STALL 的觸發條件是「電流高且滑差大」，但因為 slip_ratio 是從 current_a 估算的，兩個條件有相關性，不是真正獨立的雙條件。接上真實轉速感測器後才能真正獨立。
未來接軌真實硬體

系統架構已經為接真實硬體做好準備：

換真實硬體：只改 comms.py（實作 Modbus RTU）
換不同規格馬達：只改 config.py（銘牌規格）
重新訓練 ML：用真實資料替換模擬訓練資料
校正 Rules 閾值：根據真實保護繼電器設定值調整
升級 BEARING_WEAR：加裝振動感測器 + FFT

核心的雙層判斷架構、物理計算層、資料流設計都不需要改動。
